# Formatter Checklist

## Scope

Apply this rule to artifacts generated by AI, whether they were written directly or produced through an intermediate step.

## Formatter Selection

1. Prefer an explicitly requested formatter when one exists.
2. Otherwise use the VS Code default formatter configured for the file type.
3. If the file type has no configured formatter, use a safe built-in or repository-standard formatter only when it is clearly appropriate.

## Timing

1. Format after the functional edit is complete.
2. Format before final tests, diff review, and handoff.
3. Re-run verification if formatting changes executable or structured files.

## Practical Rules

1. Format only the files affected by the task unless a broader cleanup was requested.
2. Avoid formatting binary files or generated assets that would be damaged by text formatters.
3. Preserve user-owned unrelated changes.
4. Mention skipped files when formatting was intentionally not applied.

## Formatter Discovery

When no formatter is explicitly requested, check these sources in order:

1. `.vscode/settings.json` → `[<language>].editor.defaultFormatter`
2. Project config files: `.prettierrc`, `biome.json`, `.eslintrc`
3. VS Code extension recommendations in `.vscode/extensions.json`
