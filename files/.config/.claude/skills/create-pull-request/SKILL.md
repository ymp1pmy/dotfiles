# プルリクエスト作成

このスキルは、作業ブランチから default ブランチへのプルリクエストを準備するときに使う。
プロジェクト固有の規約は必ず `CONTRIBUTING.md` から読み取り、ここに書かれたデフォルトより優先する。

## 作業フロー

1. `CONTRIBUTING.md` を読み、ブランチ命名規則・PR テンプレート・マージ条件を確認する。
2. 現在の作業ブランチ名から PR タイトルに必要な値を推定する。
3. base は default ブランチ、head は現在の作業ブランチに固定する。
4. レビュワー向けに必要な事実を整理する。
   - 変更の目的
   - 主に触れた機能やファイル
   - 実施済みの確認内容
   - 既知のリスク、後続対応、未確認事項
5. 先にタイトルを決め、そのあと本文を作る。
6. 提示または PR 作成前に、リポジトリ規約との整合を確認する。

## タイトル規則

`CONTRIBUTING.md` にタイトル形式が定義されていればそれを使う。
なければ次のデフォルトを使う。

```text
{type}: {summary}
```

type の例: `feat`, `fix`, `refactor`, `docs`, `chore`

## 本文作成

`CONTRIBUTING.md` に PR テンプレートがあればそれを使う。
テンプレートが見当たらない場合は次の 4 セクション構成を既定形として使う。

```markdown
## 概要

- 変更の目的を 1-2 点でまとめる

## 変更内容

- レビューで見てほしい変更を列挙する

## 関連 Issue

## 備考

- 実施した確認内容、リスク、未確認事項、後続対応があれば書く
```

本文ルール:

- 小さい変更では本文も短く保つ。
- 実施した確認内容や未解決のリスクは `備考` にまとめ、実行していない確認やテストは書かない。

## 作成前チェック

PR を開く前に `CONTRIBUTING.md` の条件を確認する。
`CONTRIBUTING.md` がない場合は次の一般的な条件を使う。

- 作業ブランチの変更は PR 経由で default ブランチに入れる。
- コンフリクトは作業ブランチ上で解消し、PR 作成前に挙動確認する。
- CI のテストに合格した PR だけをマージする。

未確認の項目がある場合は、PR 草案または引き継ぎで明確に書く。

## 出力内容

このスキルを使うときは、次をまとめて出す。

- PR 種別: 作業ブランチから default ブランチ
- 提案タイトル
- 提案本文
- base と head のブランチ
- `備考` に含めた確認内容と未確認事項の要約
- 安全に草案を作れない場合だけ確認事項
